**Test 1 — First-launch onboarding (APP-07):**
1. Delete existing settings: `rm ~/Library/Application\ Support/de.speakly.app/settings.json` (macOS) or `%APPDATA%\de.speakly.app\settings.json` (Windows)
2. Start app → within 1 second a Settings window should open showing Step 1 (Welcome + hotkey)

**Test 3 — Settings window (no duplicate):**
11. Click gear icon in tray popup → Settings window opens (~500x600)
12. Click gear icon again → existing window focuses, NO second window opens

**Test 4 — API key validation (APP-04):**
13. In Settings: API Keys section visible with masked inputs and Testen buttons
14. Enter an invalid key like "sk-invalid123", click Testen → inline error shown (✗)
15. Enter a valid OpenAI key, click Testen → success shown (✓) [optional if key available]
16. Close Settings window → reopen → key is still present

**Test 5 — Hotkey remap (APP-02):**
17. In Settings: Hotkey section shows current hotkey
18. Click Aendern → input turns blue, shows "Tastenkombination druecken..."
19. Press e.g. Cmd+Shift+R → hotkey updates to "CommandOrControl+Shift+R"
20. Restart app → new hotkey is still set

**Test 6 — All settings persist (phase success criterion 5):**
21. Set API key + mic selection + new hotkey
22. Quit and restart app
23. Settings still show the same values
