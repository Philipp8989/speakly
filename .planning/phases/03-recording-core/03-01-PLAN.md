**capture.rs:**
```rust
// Speakly — Audio-Aufnahme via cpal
// Startet einen dedizierten Thread fuer Audio-Input.
// WICHTIG: cpal::Stream ist !Send — bleibt auf dem erzeugenden Thread.
// Kommunikation mit Hauptthread via std::sync::mpsc::Sender<Vec<f32>>.

pub fn start_capture_thread(
    device_name: Option<String>,
    sender: std::sync::mpsc::Sender<Vec<f32>>,
    stop_flag: std::sync::Arc<std::sync::atomic::AtomicBool>,
    app_handle: tauri::AppHandle,
    native_sample_rate_out: std::sync::Arc<std::sync::Mutex<u32>>,
)
```

- Use `device.default_input_config()` — NEVER hardcode channels/rate (Pitfall 3)
- Match on config.sample_format(): F32 and I16 branches
- In the cpal data callback: compute RMS of the frame, emit "audio_level" event via app_handle.emit() — rate limit to ~12Hz using frame counter modulo
- Send raw f32 frames to sender (convert i16→f32 inline if needed)
- Check stop_flag AtomicBool inside the stream loop (use a separate thread that polls stop_flag and drops the stream)
- Store native sample_rate into native_sample_rate_out on stream creation
- Function returns immediately; the stream lives in a spawned thread that keeps it alive until stop_flag is set

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS mic → cpal callback | Untrusted raw audio data enters here |
| ConfigStore → Rust code | Stored settings (silence_timeout_ms, vad_rms_threshold) read at runtime — user or file system could tamper |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | silence_timeout_ms from ConfigStore | mitigate | Clamp value to 500..=30000 ms at read time in lib.rs before passing to SilenceCounter::new(); never use raw stored value |
| T-03-02 | Tampering | vad_rms_threshold from ConfigStore | mitigate | Clamp to 0.001..=1.0 f32 range at read time; default 0.01 |
| T-03-03 | Denial of Service | Unbounded audio buffer growth | mitigate | Enforce hard cap: if audio_buffer exceeds 10 minutes of 44.1kHz f32 samples (~10*60*44100*4 ≈ 100MB), stop recording and emit recording_discarded with reason "too_long" |
| T-03-04 | Information Disclosure | WAV data written to disk | accept | D-07 mandates in-memory only; no temp file paths in wav_encoder.rs; risk accepted: WAV bytes in memory are process-private |
</threat_model>
