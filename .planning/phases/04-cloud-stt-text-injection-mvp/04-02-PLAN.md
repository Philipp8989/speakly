    ## App.tsx

    Extend the existing useEffect (non-settings branch) with:

    1. Add 'recording_stopped' listener:
       ```typescript
       listen<{ wav_ready: boolean; duration_ms: number }>('recording_stopped', (event) => {
         if (event.payload.wav_ready) {
           invoke('transcribe_and_inject').catch((err) => {
             console.error('Speakly: transcribe_and_inject Fehler:', err);
           });
         }
         // wav_ready=false means VAD discarded — do nothing (per D-24)
       }).then(ul => unlisteners.push(ul));
       ```

    2. Add 'transcription_state_changed' listener that updates appState:
       ```typescript
       listen<TranscriptionStatePayload>('transcription_state_changed', (event) => {
         const p = event.payload;
         if (p.state === 'processing') {
           setAppState({ kind: 'processing' });
         } else if (p.state === 'done') {
           setAppState({ kind: 'transcript', text: p.text });
           setTimeout(() => setAppState({ kind: 'idle' }), 2000);  // per D-21
         } else if (p.state === 'error') {
           setAppState({ kind: 'error', message: p.message });
           setTimeout(() => setAppState({ kind: 'idle' }), 5000);  // per D-22
         }
       }).then(ul => unlisteners.push(ul));
       ```

    3. Update 'recording_state_changed' listener to also set appState:
       ```typescript
       listen<{ recording: boolean }>('recording_state_changed', (event) => {
         setAppState(event.payload.recording ? { kind: 'recording' } : { kind: 'idle' });
       }).then(ul => unlisteners.push(ul));
       ```

    4. Lift appState to App component:
       ```typescript
       const [appState, setAppState] = useState<AppState>({ kind: 'idle' });
       ```
       Pass it to TrayPopup: <TrayPopup onSettingsClick={...} appState={appState} />

    5. Add TranscriptionStatePayload type import or inline type.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → Rust invoke | invoke('transcribe_and_inject') triggers audio access + external HTTP call |
| Rust events → Frontend | transcription_state_changed payload rendered directly into DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | App.tsx — setTimeout for auto-dismiss | accept | Timers are local state, not user-controllable from outside the app. No threat surface. |
| T-04-07 | Information Disclosure | TrayPopup — transcript text displayed in popup | accept | Popup is app-local, visible only to user who triggered recording. Not leaked to OS or network. |
| T-04-08 | Spoofing | transcription_state_changed event — forged by another app? | accept | Tauri IPC is process-local, not accessible by external processes. No mitigation needed. |
| T-04-09 | Information Disclosure | TrayPopup — error message may leak API error details | mitigate | Error messages from Rust are already sanitised to German user-facing strings (D-04, D-05). Frontend renders exactly what Rust sends — no raw HTTP bodies forwarded. |
</threat_model>
