<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify Phase 4 success criteria end-to-end</name>
  <what-built>
    Complete MVP pipeline:
    - Whisper API transcription (stt/mod.rs) triggered by recording_stopped event
    - Clipboard save/restore with enigo paste simulation (inject/mod.rs)
    - TrayPopup processing/transcript/error states
    - Language dropdown in SettingsPage
  </what-built>
  <how-to-verify>
    PRECONDITION: Ensure your OpenAI API key is set in Speakly Settings → API Keys.

    Test each criterion in order. Record pass/fail:

    --- Criterion 1: Text injection within 2-3 seconds ---
    1. Open any text field (browser address bar, TextEdit, Notes)
    2. Press the configured hotkey, speak "Hallo, das ist ein Test", release
    3. Watch the tray popup: it should show "Verarbeite..." (yellow dot) then briefly show the transcript
    4. PASS if: transcribed text appears in the text field within 3 seconds of releasing hotkey
    5. FAIL if: text field is empty, or takes > 3 seconds

    --- Criterion 2: Punctuation and capitalization ---
    1. Speak a sentence with natural pauses: "Heute ist Montag. Wie geht es dir?"
    2. PASS if: transcription shows correct German capitalization and a period between sentences
    3. FAIL if: all lowercase, no punctuation

    --- Criterion 3: Language setting works ---
    1. Open Speakly Settings → find the language dropdown
    2. Set language to "Englisch" (en)
    3. Record: "The weather today is quite nice"
    4. PASS if: transcription is in English (not auto-detected as German attempt)
    5. Set language back to "Automatisch erkennen"
    6. Record the same sentence — PASS if still transcribed as English (auto-detected correctly)

    --- Criterion 4: Clipboard is preserved ---
    1. Copy something to clipboard: Cmd+C a URL or a code snippet
    2. Use Speakly to dictate a short phrase (any text field)
    3. After the transcript appears in the text field, press Cmd+V in a different text field
    4. PASS if: original copied content pastes correctly (clipboard was restored)
    5. FAIL if: transcript text or empty string pastes instead of original

    --- Criterion 5: Works in multiple apps ---
    Test all four targets below and record each:
    a) Browser text field (Chrome/Safari address bar or a web form)
    b) VS Code (open a .txt file, place cursor, dictate)
    c) Slack (if available — compose field)
    d) macOS Notes (body text area)
    PASS if: text appears in field in all tested apps
    FAIL if: any app pastes nothing or shows an error

    --- Error paths (bonus, not required for PASS) ---
    - Clear API key, try to dictate → popup should show "API-Key fehlt oder ungueltig"
    - Speak silence for 1 second → popup should discard without calling API (no "Verarbeite...")

    REPORT: For each criterion write PASS or FAIL with a brief note.
  </how-to-verify>
