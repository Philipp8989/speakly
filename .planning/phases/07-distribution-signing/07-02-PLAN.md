    Create `.github/workflows/release.yml` with the following structure.
    Create the directory first: `mkdir -p .github/workflows`

    Per D-09: trigger on tag push matching `v*.*.*`
    Per D-10: matrix with macos-latest and windows-latest
    Per D-11: steps are checkout → Rust → Node/pnpm → deps → build → sign → upload
    Per D-12: secrets from GitHub repo settings

    ```yaml
    name: Release

    on:
      push:
        tags:
          - 'v[0-9]+.[0-9]+.[0-9]+'

    jobs:
      release:
        permissions:
          contents: write
        strategy:
          fail-fast: false
          matrix:
            include:
              - platform: macos-latest
                args: '--target universal-apple-darwin'
              - platform: windows-latest
                args: ''

        runs-on: ${{ matrix.platform }}

        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Install Rust stable
            uses: dtolnay/rust-toolchain@stable
            with:
              targets: ${{ matrix.platform == 'macos-latest' && 'aarch64-apple-darwin,x86_64-apple-darwin' || '' }}

          - name: Rust cache
            uses: swatinem/rust-cache@v2
            with:
              workspaces: './src-tauri -> target'

          - name: Setup Node
            uses: actions/setup-node@v4
            with:
              node-version: 20

          - name: Install pnpm
            uses: pnpm/action-setup@v4
            with:
              version: 9

          - name: Install frontend dependencies
            run: pnpm install

          - name: Import Apple certificate (macOS only)
            if: matrix.platform == 'macos-latest'
            env:
              APPLE_CERTIFICATE: ${{ secrets.APPLE_CERTIFICATE }}
              APPLE_CERTIFICATE_PASSWORD: ${{ secrets.APPLE_CERTIFICATE_PASSWORD }}
              KEYCHAIN_PASSWORD: ${{ secrets.KEYCHAIN_PASSWORD }}
            run: |
              CERTIFICATE_PATH=$RUNNER_TEMP/build_certificate.p12
              KEYCHAIN_PATH=$RUNNER_TEMP/app-signing.keychain-db
              echo -n "$APPLE_CERTIFICATE" | base64 --decode -o $CERTIFICATE_PATH
              security create-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
              security set-keychain-settings -lut 21600 $KEYCHAIN_PATH
              security unlock-keychain -p "$KEYCHAIN_PASSWORD" $KEYCHAIN_PATH
              security import $CERTIFICATE_PATH -P "$APPLE_CERTIFICATE_PASSWORD" -A -t cert -f pkcs12 -k $KEYCHAIN_PATH
              security list-keychain -d user -s $KEYCHAIN_PATH
              security find-identity -v -p codesigning

          - name: Build and release
            uses: tauri-apps/tauri-action@v0
            env:
              GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
              # macOS signing (per D-03)
              APPLE_SIGNING_IDENTITY: ${{ secrets.APPLE_SIGNING_IDENTITY }}
              # macOS notarization (per D-04, D-05)
              APPLE_ID: ${{ secrets.APPLE_ID }}
              APPLE_TEAM_ID: ${{ secrets.APPLE_TEAM_ID }}
              APPLE_PASSWORD: ${{ secrets.APPLE_PASSWORD }}
              # Windows signing (per D-07)
              TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
              TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
            with:
              tagName: ${{ github.ref_name }}
              releaseName: 'Speakly ${{ github.ref_name }}'
              releaseBody: |
                See [CHANGELOG.md](https://github.com/${{ github.repository }}/blob/main/CHANGELOG.md) for release notes.
              releaseDraft: false
              prerelease: false
              args: ${{ matrix.args }}
    ```

    Key notes for the executor:
    - `tauri-apps/tauri-action@v0` handles the full Tauri build + bundle + upload flow
    - The Apple certificate import block is a standard pattern for macOS CI signing — it creates a temporary keychain, imports the base64-encoded .p12 cert, and makes it available to codesign
    - `APPLE_SIGNING_IDENTITY` must match the Common Name of the Developer ID Application certificate (e.g. "Developer ID Application: Your Name (TEAMID)")
    - `APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_PASSWORD` are used by notarytool for notarization (per D-05)
    - `TAURI_SIGNING_PRIVATE_KEY` is the Tauri updater signing key — also used for Windows MSI signing (per D-07)
    - `KEYCHAIN_PASSWORD` is a random string used only for the temporary CI keychain — add it as a GitHub secret too
    - Per D-08: if TAURI_SIGNING_PRIVATE_KEY is not set (no Windows cert), the build will still produce an .msi but SmartScreen may warn — acceptable for internal team use

    Do NOT add Linux to the matrix (per deferred: Linux not needed).
    Do NOT add auto-update config (per deferred: no auto-update in v1).

    # GitHub Actions Secrets Setup

    Configure these secrets in: GitHub repo → Settings → Secrets and variables → Actions → New repository secret

    ## macOS Signing & Notarization

    | Secret Name | Value | How to Get |
    |-------------|-------|-----------|
    | APPLE_CERTIFICATE | Base64-encoded .p12 file | Export from Keychain Access → Developer ID Application cert → Export as .p12 → `base64 -i cert.p12` |
    | APPLE_CERTIFICATE_PASSWORD | Password set during .p12 export | Set during Keychain export |
    | KEYCHAIN_PASSWORD | Any random string (CI-only) | Generate: `openssl rand -hex 16` |
    | APPLE_SIGNING_IDENTITY | Common Name of your Developer ID cert | `security find-identity -v -p codesigning` → e.g. "Developer ID Application: Your Name (ABCD1234EF)" |
    | APPLE_ID | Your Apple ID email | The Apple ID registered with Apple Developer Program |
    | APPLE_TEAM_ID | Your Apple Developer Team ID | developer.apple.com → Membership → Team ID (10-char string) |
    | APPLE_PASSWORD | App-specific password for notarytool | appleid.apple.com → Sign In → App-Specific Passwords → Generate |

    ## Windows Signing

    | Secret Name | Value | How to Get |
    |-------------|-------|-----------|
    | TAURI_SIGNING_PRIVATE_KEY | Tauri updater private key (base64) | Generate: `pnpm tauri signer generate -w ~/.tauri/speakly.key` → outputs private key |
    | TAURI_SIGNING_PRIVATE_KEY_PASSWORD | Password for the signing key | Set during key generation above |

    ## Notes

    - macOS signing requires an **Apple Developer Program** membership ($99/year)
    - Windows signing via TAURI_SIGNING_PRIVATE_KEY is Tauri's built-in updater signing — it is NOT an Authenticode certificate. Binaries will still show SmartScreen warning on first run but will be verifiably authentic to users who trust the key. For full SmartScreen suppression, a commercial Authenticode certificate (EV) is required — acceptable cost for v2.
    - All secrets are injected at build time only — they are never stored in the repository

    ## Triggering a Release

    ```bash
    git tag v0.1.0
    git push origin v0.1.0
    ```

    This triggers the workflow. Monitor at: https://github.com/{owner}/{repo}/actions
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → signing secrets | Secrets injected as env vars; never logged or stored in artifacts |
| CI runner → Apple notarization API | notarytool submits binary over HTTPS with app-specific password |
| CI runner → GitHub Releases | GITHUB_TOKEN scoped to contents:write for this repo only |
| signed .dmg/.msi → end user | Code signing + notarization provides integrity guarantee |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Info Disclosure | CI secrets in workflow logs | mitigate | Use `${{ secrets.X }}` — GitHub Actions automatically masks secret values in logs |
| T-07-05 | Tampering | .msi without EV cert (per D-08) | accept | Tauri signing key provides authenticity for internal team; SmartScreen warning on first run is acceptable; EV cert deferred to v2 |
| T-07-06 | Spoofing | Apple Developer ID cert | mitigate | Certificate private key stored as base64 in GitHub secrets; only writable by repo admins |
| T-07-07 | Elevation of Privilege | workflow trigger on any tag | accept | Repo is private/internal; tag push requires write access; no external contributors |
| T-07-08 | Info Disclosure | APPLE_PASSWORD (app-specific) | mitigate | App-specific passwords can be revoked individually at appleid.apple.com without changing main password |
</threat_model>
