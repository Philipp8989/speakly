.github/workflows/release.yml erstellen:

Die Pipeline baut auf:
- Tag-Push (v*) — Release-Build fuer alle Plattformen
- macOS: universal-apple-darwin (arm64 + x86_64)
- Windows: x86_64-pc-windows-msvc

Signing-Secrets werden als GitHub Secrets erwartet (werden separat konfiguriert):
- APPLE_CERTIFICATE — Base64-kodiertes .p12 Zertifikat
- APPLE_CERTIFICATE_PASSWORD — Passwort fuer das Zertifikat
- APPLE_SIGNING_IDENTITY — z.B. "Developer ID Application: ..."
- APPLE_ID — Apple-ID fuer Notarization
- APPLE_PASSWORD — App-spezifisches Passwort fuer Notarization
- APPLE_TEAM_ID — Apple Developer Team ID
- TAURI_SIGNING_PRIVATE_KEY — Tauri Update-Signing Key (optional)
- WINDOWS_CERTIFICATE — Base64-kodiertes .pfx Zertifikat (optional fuer Windows-Signing)
- WINDOWS_CERTIFICATE_PASSWORD — Passwort fuer Windows-Zertifikat

Die Pipeline laeuft auch ohne Signing-Secrets (unsigned builds fuer Testing).
